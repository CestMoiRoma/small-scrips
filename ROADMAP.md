# Roadmap

Planned additions to the [`small-doc/`](small-doc/) section.
Check items off as they get done.

---

## 🔥 High priority — used constantly

- [ ] **Regex** — syntax, character classes, groups, lookaheads, a cheatsheet usable alongside every other doc
- [ ] **npm / yarn / pnpm** — the JS equivalent of pip, daily use for most devs
- [ ] **SSH** — deserves its own dedicated doc (keys, config file, tunnels, agent forwarding, jump hosts)
- [ ] **curl** — deep coverage of auth, REST calls, headers, file upload, response inspection, timing
- [ ] **Vim basics** — insert/normal/visual modes, save/quit, search/replace, common shortcuts (everyone gets stuck in it)
- [ ] **HTTP status codes** — fast reference for 2xx / 3xx / 4xx / 5xx with descriptions and use cases

---

## 🟡 Good second tier

- [ ] **Makefile** — still the most common build glue across languages and projects
- [ ] **jq** — de facto standard for CLI JSON processing
- [ ] **Semantic Versioning (SemVer)** — `MAJOR.MINOR.PATCH` rules, version ranges, pre-release syntax
- [ ] **GitHub Actions** — CI/CD workflow syntax, triggers, jobs, steps, secrets, matrix builds
- [ ] **YAML** — syntax reference (indentation, types, anchors, multiline) — used in Compose, Actions, K8s
- [ ] **SQL basics** — SELECT, JOIN, WHERE, GROUP BY, subqueries, common patterns

---

## 🔵 Further down the line

- [ ] **kubectl / Kubernetes** — for running containers in production
- [ ] **tmux** — session management, panes, windows, keybindings
- [ ] **License types** — MIT, Apache 2.0, GPL, AGPL — what each allows and requires
- [ ] **GPG** — key generation, signing commits, encrypting/decrypting files
